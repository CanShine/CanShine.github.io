2. 个人笔记

2.1 一般工作完整流程

（新增或更新 Hexo 博客并部署上线）

  1. 新建或编辑文章
    在 source/_posts/ 目录下新建 Markdown 文件，或修改已有文章。

  2. 清理旧文件

    清除缓存和旧的生成文件，避免异常。

  3. 生成静态页面

    根据 Markdown 文章生成静态网页文件。

  4. 本地预览（可选）

    启动本地服务器，访问http://localhost:4000检查效果。

  5. 部署到线上

    将生成的静态文件上传到配置好的服务器（如 GitHub Pages）。

完整流程总结
新建/编辑文章 → hexo clean → hexo g → hexo s（预览）→ hexo d（部署上线）

2.2 补充说明

  • 重新部署博客内容后，如若网页内容未更新，则需要清除浏览器缓存，可以使用以下快捷键：Ctrl + Shift + R
  • 标题的日期不能重复，否则会导致文章无法正常显示。

3. _config.yml文件介绍

1. 站点信息（Site）

  • title：站点标题。
  • subtitle：副标题。
  • description：站点描述。
  • keywords：关键词。
  • author：作者名。
  • language：语言（如 zh-CN 表示中文）。
  • timezone：时区。

2. 网址设置（URL）

  • url：你博客的主网址（如 GitHub Pages 填你的主页地址）。
  • permalink：文章永久链接格式。
  • pretty_urls：美化 URL，去掉 index.html 或 .html。

3. 目录设置（Directory）

  • 各种目录的名称，如 source_dir（源码目录）、public_dir（生成目录）等。

4. 写作相关（Writing）

  • new_post_name：新文章的默认文件名格式。
  • default_layout：默认布局。
  • external_link：外链是否新窗口打开等。
  • syntax_highlighter：代码高亮方式（如 highlight.js）。

5. 首页设置（Home page setting）

  • index_generator：首页文章分页、排序等设置。

6. 分类与标签（Category & Tag）

  • default_category：默认分类。
  • category_map、tag_map：分类和标签映射。

7. 元数据（Metadata elements）

  • meta_generator：是否生成 meta 标签。

8. 时间格式（Date / Time format）

  • date_format、time_format：日期和时间显示格式。
  • updated_option：更新时间的依据。

9. 分页（Pagination）

  • per_page：每页显示文章数。
  • pagination_dir：分页目录名。

10. 包含/排除文件（Include / Exclude file(s)）

  • include、exclude、ignore：指定哪些文件包含或排除。

11. 扩展（Extensions）

  • theme：主题名称。
  • 插件、主题相关配置。

12. 部署（Deployment）

  • deploy：部署方式，这里用 git，将生成的静态文件推送到 GitHub Pages。
